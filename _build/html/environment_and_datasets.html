Environment and Data Sets#

Environment#

For the practical part of our workshop, we would like to introduce you to our environment:

  • CentOS Linux

  • module system for using our software:

    • Python 3.8.8 with many geo scientific libraries:

      • module load anaconda3

      • spyder (IDE)

      • jupyter notebook (Jupyter Notebook)

      • python (interpreter)

    • R 4.1.1:

      • module load r

    • ncdump can be used without loading modules

  • You have 20GB space in your home directory. This should be enough for the test cases. If you need more, get back to us.

Data Sets#

You can use the following ASCII or binary data sets to practice the conversion to CF compliant netCDF files:

  1. DWD stations We have atmospheric measurement stations from the German Meteorological Service (DWD) in ASCII: /data/icdc/atmosphere/dwd_stations/DATA/

You can either convert single stations to netCDF, but you can also try to save several stations into one netCDF file!

  1. Radolan You can also convert a binary data sets, the same one you saw in the R part, but now in a higher temporal resolution: https://opendata.dwd.de/climate_environment/CDC/grids_germany/5_minutes/radolan/reproc/2017_002/bin/2023/
